# Развертывание и использование в Docker

Система полностью контейнеризирована с использованием **Docker Compose**. Это позволяет запустить ядро анализатора и базу данных PostgreSQL в изолированной среде одной командой.

## 1. Подготовка
Перед запуском убедитесь, что у вас установлены:
- Docker
- Docker Compose

Скопируйте настройки из шаблона и заполните их (особенно пароль БД):

```bash
cp .env.example .env
```

## 2. Запуск инфраструктуры
Для сборки образов и запуска всех сервисов (API + БД) выполните:

```bash
docker-compose up -d --build
```

## 3. Использование CLI (Терминал)
Несмотря на то, что анализатор запущен в контейнере, вы можете проверять локальные файлы через командную строку. 

Для анализа файла выполните команду docker exec:

```
ash
docker exec -it analyzer_app_cont python main.py student_lab.py
```

## 4. Использование API
После запуска через Docker Compose, API автоматически становится доступно на порту 8000.

Интерактивная документация (Swagger): http://localhost:8000/docs
Эндпоинт анализа: POST http://localhost:8000/analyze

Пример запроса через cURL:

```Bash
curl -X POST "http://localhost:8000/analyze" \
     -H "Content-Type: application/json" \
     -d '{"project_name": "test", "code": "def func():\n    pass"}'
```

## 5. Доступ к базе данных
Внутри сети Docker база данных работает на стандартном порту 5432.
Однако для доступа с вашего компьютера (например, через pgAdmin) порт проброшен как 5433, чтобы избежать конфликтов с локально установленным PostgreSQL.

Параметры подключения для внешних инструментов:

- Host: localhost
- Port: 5433
- User/Pass: из вашего файла .env
